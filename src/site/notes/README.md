# Obsidian Vault 结构指南

## 目录概览

```
obsidian-yj/
├── 01-Daily/          # 日记、周报、会议记录、临时笔记
├── 02-Projects/       # 进行中的项目（有明确截止日期）
├── 03-Areas/          # 持续关注的领域（无截止日期）
├── 04-Notes/          # 核心知识库、原子笔记、Zettel
├── 05-Resources/      # 参考资料、书籍笔记、文章摘录
├── 06-Templates/      # 笔记模板
├── 07-Attachments/    # 图表、图片、PDF等媒体文件
├── 08-Archive/        # 已完成项目、归档内容
└── 09-Meta/           # 保险库配置、组织规则说明
```

## 各文件夹说明

### 01-Daily
- **用途**：日常记录、灵感捕捉、临时笔记
- **特点**：按时间组织，快速收集
- **示例**：Daily Note、会议记录、想法记录

### 02-Projects
- **用途**：有明确目标和截止日期的项目
- **特点**：项目级别的组织，包含子任务和进度
- **示例**：路由补丁项目、开发任务

### 03-Areas
- **用途**：持续关注的生活或工作领域
- **特点**：无明确截止日期，长期维护
- **示例**：学习、健康、职业发展

### 04-Notes
- **用途**：核心知识库，原子笔记
- **特点**：高度互联，支持知识图谱
- **示例**：概念笔记、研究笔记、技术文档

### 05-Resources
- **用途**：外部参考资料和信息源
- **特点**：被动收集，用于查阅
- **示例**：书籍摘录、文章链接、参考资料

### 06-Templates
- **用途**：可复用的笔记模板
- **特点**：提高创建新笔记的效率
- **示例**：Daily Note Template、Weekly Note、Books

### 07-Attachments
- **用途**：所有媒体文件和图表
- **特点**：包含Excalidraw图表
- **示例**：架构设计图、流程图、截图

### 08-Archive
- **用途**：已完成或不再活跃的内容
- **特点**：保留历史记录，便于查阅
- **示例**：已完成项目、过期资料

### 09-Meta
- **用途**：保险库管理和配置
- **特点**：系统级别的文档
- **示例**：组织规则、插件配置说明

## 使用建议

### 标签系统
使用嵌套标签进行多维分类：
- `#project/项目名` - 项目标记
- `#area/领域名` - 领域标记
- `#status/active` - 状态标记
- `#type/concept` - 笔记类型

### 属性元数据
在笔记frontmatter中添加：
```yaml
---
status: active
priority: high
date: 2026-03-22
tags: [project/xxx]
---
```

### 导航中心
在各文件夹创建 `_Index.md` 作为导航中心，链接到该文件夹的重要笔记。

## 工作流建议

1. **快速捕捉** → 01-Daily/Inbox
2. **处理整理** → 按类型移至相应文件夹
3. **建立链接** → 使用内部链接和标签
4. **定期回顾** → 通过MOC和搜索发现连接
5. **归档完成** → 移至08-Archive
